# Fabric Report Load Balancer

This notebook is meant to be run locally and has exactly the same functionality as the notebook that is meant to run inside Fabric. 

For setup you just need to change whatever settings the first cell of the notebook asks for. When running locally you need to provide SQL Connection string from a fabric warehouse that contains the telemetry produced by the script which helps you understand what happens in each iteration.

requirements.txt contains a list of dependencies that the notebook runs locally via conda python environment.
